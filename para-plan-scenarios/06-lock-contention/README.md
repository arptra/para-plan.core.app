# Конкуренция за блокировки

**Что не так:** Долгая транзакция держит блок, другие сессии висят.

**Как проявляется:** SELECT ... FOR UPDATE зависает/таймаут.

**Что обнаружит анализатор:** Совет: найти и завершить blocker; избегать idle-in-transaction.

**Исправление:** pg_terminate_backend() или корректный COMMIT/ROLLBACK.

## Как запустить
```bash
docker-compose up -d
./demo.sh
```
