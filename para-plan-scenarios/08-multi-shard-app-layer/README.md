# Имитация шардинга на уровне приложения

**Что не так:** Клиент объединяет данные из разных шардов и сортирует без pushdown.

**Как проявляется:** Большие выборки, дублирование работы.

**Что обнаружит анализатор:** Советы уровня архитектуры: pre-aggregation per shard, реплики для чтения.

**Исправление:** Материализованные агрегаты и объединение малых результатов на клиенте.

## Как запустить
```bash
docker compose up -d
./demo.sh
```
