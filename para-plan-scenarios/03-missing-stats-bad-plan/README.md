# Нет статистики после bulk insert → плохой план

**Что не так:** Планировщик не знает распределение данных.

**Как проявляется:** Нереалистичные оценочные rows и неоптимальный план.

**Что обнаружит анализатор:** Совет STATS/ANALYZE; может предложить индекс t(a,b).

**Исправление:** Выполнить ANALYZE (и при необходимости добавить индекс).

## Как запустить
```bash
docker compose up -d
./demo.sh
```
