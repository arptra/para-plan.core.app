# Слишком низкие work_mem / effective_cache_size

**Что не так:** Сортировки/агрегации проливаются на диск, планировщик недооценивает кэш.

**Как проявляется:** EXPLAIN показывает Disk в Sort/HashAgg и медленную работу.

**Что обнаружит анализатор:** Совет CONFIG: поднять work_mem/ effective_cache_size, индекс по фильтру.

**Исправление:** Поднять параметры и/или добавить индекс.

## Как запустить
```bash
docker compose up -d
./demo.sh
```
